# Using ATLAS on Wahab

Downstream analysis was run using [ATLAS](https://atlaswiki.netlify.app/). Scripts are adapted from the Wegmann Lab. Consider this a best-practice guide, although the best practice is currently in flux.

<details><summary>1. Pre-processing</summary>

### 1. Pre-processing

ATLAS uses mapped .bam & .bai files as well as GERP scores, which can both be generated by GenErode. Run GenErode first to obtain these files. As long as the .bam & .bai files and GERP scores are generated GenErode does not need to fully complete in order to move on to the ATLAS step.

Check GenErode output files, which are ATLAS input files: 
* GERP scores from the `\*.ancestral.rates.gz` file in `GenErode_Spp_20k/results/gerp`
* Modern `\*merged.rmdup.merged.realn.bam` & `\*merged.rmdup.merged.realn.bai` files in `GenErode_Spp_20k/results/modern/mapping/reference.denovoSSL.Spp20k/`
* Historical `\*merged.rmdup.merged.realn.bam` & `\*merged.rmdup.merged.realn.bai` files in `GenErode_Spp_20k/results/historical/mapping/reference.denovoSSL.Spp20k/`

Make an ATLAS_Spp directory in your species directory with your species code. 
```
cd /archive/carpenterlab/pire/pire_genus_species_lcwgs

mkdir ATLAS_Spp
```

Make ATLAS subdirectories.
```
cd ATLAS_Spp

mkdir beds recal theta plots scripts logs
```

Copy ATLAS scripts to the ATLAS_Spp directory. 
```
cp /archive/carpenterlab/pire/pire_lcwgs_data_processing/scripts/ATLAS_wahab/*.sbatch .

cp /archive/carpenterlab/pire/pire_lcwgs_data_processing/scripts/ATLAS_wahab/*.bash .

cp /archive/carpenterlab/pire/pire_lcwgs_data_processing/scripts/ATLAS_wahab/*.R .
```

</details>


<details><summary>2. Binning GERP scores</summary>

### 2. Binning GERP scores

To generate bed files based on binned GERP scores for ATLAS, you will have to process the `reference.denovoSSL.Spp20k.ancestral.rates.gz` file generated by GenErode. So far we have used three bins (sites with scores >1.5, sites with scores between 1.0 and 1.5, and sites with scores between 0.5 and 1).  
```
# move to your species ATLAS directory

cd /archive/carpenterlab/pire/pire_genus_species_lcwgs/ATLAS_Spp

# get sites with scores in each particular bin using the reference.denovoSSL.Spp20k.ancestral.rates.gz file from GenErode

gunzip -c /archive/carpenterlab/pire/pire_genus_species_lcwgs/GenErode_Spp_20k/results/gerp/reference.denovoSSL.Spp20k.ancestral.rates.gz | awk -v OFS='\t' '($4 >= 1.5) {print $1,$2,$2}' > gerp_15.bed 

gunzip -c /archive/carpenterlab/pire/pire_genus_species_lcwgs/GenErode_Spp_20k/results/gerp/reference.denovoSSL.Spp20k.ancestral.rates.gz | awk -v OFS='\t' '($4 >= 1 && $4 < 1.5) {print $1,$2,$2}' > gerp_10.bed 

gunzip -c /archive/carpenterlab/pire/pire_genus_species_lcwgs/GenErode_Spp_20k/results/gerp/reference.denovoSSL.Spp20k.ancestral.rates.gz | awk -v OFS='\t' '($4 >= 0.5 && $4 < 1) {print $1,$2,$2}' > gerp_05.bed 
```

Check the number of sites in your `gerp_\*.bed` files. ATLAS wants something on the order of ~5 million total sites for its recalibration procedure. If the default bins do not generate ~5 million total sites, then adjust the lower bin threshhold of 0.5 down to capture more sites. You can move forward with just two bins if necessary. 
```
wc -l gerp_15.bed

wc -l gerp_10.bed 

wc -l gerp_05.bed 
```

Merge adjacent sites into regions using bedtools.
```
module load bedtools

crun bedtools merge -i gerp_15.bed > gerp_15.merge.bed 

crun bedtools merge -i gerp_10.bed > gerp_10.merge.bed 

crun bedtools merge -i gerp_05.bed > gerp_05.merge.bed 
```

Remove singleton sites from `gerp_\*.merge.bed` files.
```
awk -v OFS='\t' '!($2==$3)' < gerp_15.merge.bed > gerp_15.merge.nosingle.bed 

awk -v OFS='\t' '!($2==$3)' < gerp_10.merge.bed > gerp_10.merge.nosingle.bed 

awk -v OFS='\t' '!($2==$3)' < gerp_05.merge.bed > gerp_05.merge.nosingle.bed 
```

Copy all .bed files to the recal directory, which is the output directory for the recalibration step. 
```
cp *.bed recal
```

</details>


<details><summary>3. Run Recalibration</summary>

### 3. Run Recalibration

ATLAS has its own recalibration procedure, so you will use the '\*merged.rmdup.merged.realn.bam' files produced by GenErode before it performs its recalibration. This step will have to be run separately on the modern and historical files, unless they are all in the same directory. 

Modify the last line of the `atlas_recal_readuntilbeds_array.bash` script to reflect the path for the `atlas_recal_readuntilbeds_array.sbatch` script.
```
/archive/carpenterlab/pire/pire_genus_species_lcwgs/ATLAS_Spp/atlas_recal_readuntilbeds_array.sbatch ${BAMDIR} ${REFLOC} ${OUTDIR})
```

Modify the last line of the `atlas_recal_readuntilbeds_array.sbatch` script to reflect the file names of the `gerp_*.merge.nosingle.bed` files you want to use. You can also change the maximum depth - currently set to only use 10 reads maximum per site, which should be sufficient to calculate error.
```
crun.atlas atlas --task estimateErrors --bam ${sample_name}.merged.rmdup.merged.realn.bam --fasta ${REFLOC} --regions gerp_15.merge.nosingle.bed,gerp_10.merge.nosingle.bed,gerp_05.merge.nosingle.bed --readUpToDepth 10
```

Check the outputs in the recal directory as the files run. Currently, ATLAS can get hung up on some files, particularly high-depth files and contemporary sequencing data. If there is no change in the \*.out log file for a long time you might want to cancel the job and start it again.

Run ATLAS recalibration with `atlas_recal_readuntilbeds_array.bash` on modern samples.
```
# change directory to your ATLAS_Spp directory
cd /archive/carpenterlab/pire/pire_genus_species_lcwgs/ATLAS_Spp

bash atlas_recal_readuntilbeds_array.bash [directory with .bam files] [reference genome location w/ filename] [output directory]

# modern run
bash atlas_recal_readuntilbeds_array.bash /archive/carpenterlab/pire/pire_species_genus_lcwgs/GenErode_Spp_20k/results/modern/mapping/reference.denovoSSL.Spp20k /archive/carpenterlab/pire/pire_species_genus_lcwgs/GenErode_Spp_20k/reference/reference.denovoSSL.Spp20k.fasta  /archive/carpenterlab/pire/pire_species_genus_lcwgs/ATLAS_Spp/recal
```

Run ATLAS recalibration with `atlas_recal_readuntilbeds_array.bash` on historical samples.
```
bash atlas_recal_readuntilbeds_array.bash [directory with .bam files] [reference genome location w/ filename] [output directory]

# historical run
bash atlas_recal_readuntilbeds_array.bash /archive/carpenterlab/pire/pire_genus_species_lcwgs/GenErode_Spp_20k/results/historical/mapping/reference.denovoSSL.Spp20k /archive/carpenterlab/pire/pire_genus_species_lcwgs/GenErode_Spp_20k/reference/reference.denovoSSL.Spp20k.fasta  /archive/carpenterlab/pire/pire_genus_species_lcwgs/ATLAS_Spp/recal
```

<details><summary>5. Run Theta</summary>

### 5. Run Theta

The ATLAS theta step will estimate theta for both historical and modern samples using the whole `\*merged.rmdup.merged.realn.bam` files from the recalibration step. It also performs downsampling. This step also requires the `\*.merged.rmdup.merged.realn_RGInfo.json` files as input. 

Modify the last line in the `atlas_theta_albrecal_array.bash` script to reflect the path for the `atlas_theta_albrecal_array.sbatch` script.
```
nano atlas_theta_albrecal_array.bash

/archive/carpenterlab/pire/pire_genus_species_lcwgs/ATLAS_Spp/atlas_theta_albrecal_array.sbatch ${BAMDIR} ${OUTDIR})
```

Run ATLAS theta with `atlas_theta_albrecal_array.bash`. 
```
# change directory to your ATLAS_Spp directory
cd /archive/carpenterlab/pire/pire_genus_species_lcwgs/ATLAS_Spp

bash atlas_theta_albrecal_array.bash [directory with recalibrated output files] [output directory]

bash atlas_theta_albrecal_array.bash /archive/carpenterlab/pire/pire_genus_species_lcwgs/ATLAS_Spp/recal /archive/carpenterlab/pire/pire_genus_species_lcwgs/ATLAS_Spp/theta
```

</details>

<details><summary>8. Wrangle & Visualize Theta</summary>

### 8. Wrangle & Visualize Theta

The `wrangle_plot_theta_template.R` script wrangles and visualizes the `\*theta.txt.gz` output files from the ATLAS theta step. These files should be in the `ATLAS_Spp/theta` directory and are the input files for this script. This can be run on your personal computer by downloading the `\*theta.txt.gz` files. Alternatively, this script can be run through the RStudio Server on [ODU RCC OnDemand](https://ondemand.wahab.hpc.odu.edu/). If you use OnDemand, create an RStudio interactive session and navigate to this directory (`ATLAS_Spp/scripts`).

Rename `wrangle_plot_theta_template.R` to include your species code `wrangle_plot_theta_Spp.R`.
```
# change directory to your scripts directory
cd /archive/carpenterlab/pire/pire_genus_species_lcwgs/ATLAS_Spp/scripts

mv wrangle_plot_theta_template.R wrangle_plot_theta_Spp.R
```

In RStudio, change the user-defined variables. 
```
#### USER DEFINED VARIABLES ####

# specify input directory path with the *theta.txt.gz files and this script 
# replace the string in quotes with the absolute path of your directory
inDir = "/archive/carpenterlab/pire/pire_genus_species_lcwgs/ATLAS_Spp/theta"

# specify output directory path for plots
outDir = "/archive/carpenterlab/pire/pire_genus_species_lcwgs/ATLAS_Spp/plots"

# change your spp_code (e.g. Sob, Aen, Pbb)
spp_code="Spp"

# change your site_code (e.g. Pnd, Gal, Mvi)
site_code="Site"
```

Run through the script to generate the plots and tables. This script should be editted to evaluate your dataset when necessary.
